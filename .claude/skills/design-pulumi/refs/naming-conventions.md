# Naming Conventions for Pulumi Resources

## Two Types of Names

### 1. Logical Name (Pulumi Internal)

The first argument to every resource constructor. Used by Pulumi for:







- State tracking and identification
- URN generation
- Diff calculation
- Dependency resolution

```python
# "web-server" is the logical name

instance = aws.ec2.Instance("web-server", ...)

```



### 2. Physical Name (Cloud Provider)



The actual name in AWS/Azure/GCP. May be:

- Auto-generated by Pulumi (recommended)
- Explicitly set via resource properties

```python
# Auto-named: web-server-a1b2c3d
bucket = aws.s3.BucketV2("data-bucket")

# Explicitly named: my-company-data-prod
bucket = aws.s3.BucketV2("data-bucket", bucket="my-company-data-prod")
```

---

## Auto-Naming (Recommended Default)

### Benefits

| Benefit | Explanation |
|---------|-------------|
| No collisions | Random suffix prevents name conflicts |
| Zero-downtime updates | Create-before-delete replacement works |
| Stack isolation | Same code works across dev/staging/prod |

| Refactoring safety | Can rename logical names without recreating |


### Auto-Naming Configuration (Pulumi 3.x+)


```yaml
# Pulumi.yaml

config:
  pulumi:autonaming:

    pattern: ${project}-${stack}-${name}-${alphanum(4)}
```


Available tokens:

- `${name}` - Logical resource name
- `${project}` - Pulumi project name
- `${stack}` - Pulumi stack name
- `${organization}` - Pulumi organization
- `${alphanum(n)}` - Random alphanumeric (n chars)
- `${hex(n)}` - Random hexadecimal (n chars)

### Per-Provider Patterns

```yaml
# Pulumi.yaml
providers:
  aws:
    resources:
      "aws:s3/bucket:Bucket":
        pattern: ${name}-${stack}-${alphanum(6)}
      "aws:lambda/function:Function":
        pattern: ${project}-${name}
```

## Explicit Naming (When Required)

### Valid Use Cases

| Scenario | Reason |
|----------|--------|
| Matching existing infrastructure | Import requires exact name |
| DNS/URL requirements | CloudFront, API Gateway domains |
| Cross-account references | S3 bucket policies, IAM roles |
| External system integration | Hardcoded references |
| Compliance requirements | Naming standards |

### Explicit Naming Pattern

```python
# When explicit name is required
bucket = aws.s3.BucketV2("data-bucket",
    bucket="acmecorp-data-prod-us-west-2",  # Explicit AWS name
    opts=pulumi.ResourceOptions(
        delete_before_replace=True  # IMPORTANT: Prevents name collision
    )
)
```

### ⚠️ Explicit Naming Risks

```python
# DANGER: Explicit name without delete_before_replace
role = aws.iam.Role("app-role",
    name="my-app-role"  # Will fail on replacement!
)

# SAFE: With delete_before_replace
role = aws.iam.Role("app-role",
    name="my-app-role",
    opts=pulumi.ResourceOptions(delete_before_replace=True)
)
```

## Logical Name Conventions

### Format: kebab-case

```python
# ✅ Good
vpc = ec2.Vpc("production-vpc", ...)
bucket = s3.BucketV2("user-uploads", ...)
function = lambda_.Function("order-processor", ...)

# ❌ Bad
vpc = ec2.Vpc("ProductionVPC", ...)  # PascalCase
bucket = s3.BucketV2("user_uploads", ...)  # snake_case
function = lambda_.Function("orderprocessor", ...)  # No separator
```

### Include Purpose

```python
# ✅ Descriptive
api_gateway = apigateway.RestApi("orders-api", ...)
user_table = dynamodb.Table("users-table", ...)
auth_function = lambda_.Function("auth-handler", ...)

# ❌ Generic
api_gateway = apigateway.RestApi("api", ...)
user_table = dynamodb.Table("table", ...)
auth_function = lambda_.Function("function", ...)
```

### Component Resource Naming

```python
# In components, prefix child resources with parent name
class FargateService(pulumi.ComponentResource):
    def __init__(self, name: str, ...):
        self.task_def = ecs.TaskDefinition(f"{name}-task", ...)
        self.service = ecs.Service(f"{name}-service", ...)
        self.log_group = cloudwatch.LogGroup(f"{name}-logs", ...)
```

## Naming Utility Module

```python
# utils/naming.py
import pulumi
import hashlib
from typing import Optional

def get_resource_name(
    base_name: str,
    include_stack: bool = True,
    include_project: bool = False
) -> str:
    """Generate consistent resource names."""
    parts = []

    if include_project:
        parts.append(pulumi.get_project())
    if include_stack:
        parts.append(pulumi.get_stack())
    parts.append(base_name)

    return "-".join(parts)


def get_unique_name(
    base_name: str,
    max_length: int = 63,
    hash_length: int = 6
) -> str:
    """
    Generate unique name within length constraints.

    Useful for resources with length limits (e.g., IAM roles: 64 chars).
    """
    name = get_resource_name(base_name)

    if len(name) <= max_length:
        return name

    # Truncate and add hash for uniqueness
    hash_suffix = hashlib.md5(name.encode()).hexdigest()[:hash_length]
    truncated = name[:max_length - hash_length - 1]
    return f"{truncated}-{hash_suffix}"


def get_dns_safe_name(base_name: str) -> str:
    """Generate DNS-compliant name (lowercase, alphanumeric, hyphens)."""
    name = get_resource_name(base_name)
    # Replace underscores with hyphens, lowercase
    safe_name = name.lower().replace("_", "-")
    # Remove invalid characters
    safe_name = "".join(c for c in safe_name if c.isalnum() or c == "-")
    # Remove leading/trailing hyphens
    return safe_name.strip("-")
```

## Tagging for Discovery

When physical names are auto-generated, use tags for discovery:

```python
bucket = aws.s3.BucketV2("data-bucket",
    tags={
        "Name": "data-bucket",  # Human-readable name
        "Purpose": "user-uploads",
        "Environment": pulumi.get_stack(),
        "Project": pulumi.get_project(),
        "ManagedBy": "pulumi",
    }
)
```

## Resource Length Limits

| AWS Resource | Max Length | Recommendation |
|--------------|------------|----------------|
| S3 Bucket | 63 | Use `get_unique_name(max_length=63)` |
| IAM Role | 64 | Use `get_unique_name(max_length=64)` |
| Lambda Function | 64 | Use `get_unique_name(max_length=64)` |
| SQS Queue | 80 | Usually not a concern |
| DynamoDB Table | 255 | Usually not a concern |
| CloudWatch Log Group | 512 | Usually not a concern |

## Refactoring Names

### Renaming Logical Names

Use `aliases` to rename without recreating:

```python
# Before
bucket = aws.s3.BucketV2("old-bucket-name", ...)

# After (with alias)
bucket = aws.s3.BucketV2("new-bucket-name",
    opts=pulumi.ResourceOptions(
        aliases=[pulumi.Alias(name="old-bucket-name")]
    )
)
```

### Moving Resources Between Modules

```python
# Before: resources/storage.py
bucket = aws.s3.BucketV2("data-bucket", ...)

# After: components/secure_bucket.py (with alias)
bucket = aws.s3.BucketV2("data-bucket",
    opts=pulumi.ResourceOptions(
        aliases=[
            pulumi.Alias(
                name="data-bucket",
                parent=None  # Was at stack root
            )
        ]
    )
)
```

## Naming Checklist

- [ ] Logical names use kebab-case
- [ ] Logical names are descriptive of purpose
- [ ] Auto-naming used unless explicit name required
- [ ] Explicit names use `delete_before_replace`
- [ ] Component children prefixed with parent name
- [ ] Tags include human-readable Name
- [ ] Naming utility handles length limits
- [ ] Aliases used for refactoring
